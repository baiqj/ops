opsutils¶

opsutils is a collection of convenience functions intended to augment the built-in Python os and shutil modules.

API¶

opsutils.chmod(path, mode=None, user=None, group=None, other=None, recursive=False)¶

Changes file mode bits.

Examples:
>>> chmod('/tmp/one', 0755)

NOTE: The precending 0 is required when using a numerical mode.

opsutils.chown(path, **kwargs)¶

Change file owner and group.

Examples:
>>> chown('/tmp/one', user='root', group='apache')
opsutils.cp(src_path, dst_path, follow_links=False, recursive=True)¶

Copy source to destination.

Examples:
>>> cp('/tmp/one', '/tmp/two')
opsutils.dirs(no_class=False)¶

Returns a reference to the directory stack used by pushd and popd.

Examples:
>>> pushd('/tmp')
>>> dirs()
opsutils.exit(code=0, text='')¶

Exit and print text (if defined) to stderr if code > 0 or stdout otherwise.

Examples:
>>> exit(code=1, text='Invalid directory path')
opsutils.find(path, no_peek=False, top_down=False)¶

Find directories and files in the specified path.

Examples:
>>> for path in find('/tmp').filter(name='*.py', file=True).exclude(mtime__day=13):
...     print path
opsutils.group(id=None, name=None)¶

Get information about a group.

Examples:
>>> print group(id=0).name
root
>>> print group(name='root').id
0
opsutils.mkdir(path, recursive=True)¶

Create a directory at the specified path. By default this function recursively creates the path.

Examples:
>>> mkdir('/tmp/one/two')
opsutils.mode(value=None, user=None, group=None, other=None)¶

An object for representing file mode bits.

Example:
>>> print mode(0755).user.read
True
>>> print mode(0755).group.execute
True
>>> print mode(0755).other.write
False
opsutils.objectify()¶

Access dict-like objects via getattr.

Examples:
>>> objectify({'name': 'test'}).name
test
opsutils.path(cls, value=None, stat=None, root=None, name=None)¶

An object for representing paths.

Examples:
>>> path('/tmp').stat.owner.name
root
opsutils.popd(no_class=False)¶

Remove last path from the stack and make it the current working directory. By default popd will look for the stack variable in self if in a method and the local scope if in a function. The class behaviour can be disabled by passing no_class=True.

Examples:
>>> pushd('/tmp')
>>> popd()
opsutils.pushd(path, no_class=False)¶

Add the current working directory to the stack and switch to the path specified. By default pushd will attach the the stack variable to self if in a method and the local scope if in a function. The class behaviour can be disabled by passing no_class=True.

Examples:
>>> pushd('/tmp')
>>> popd()
opsutils.rm(path, recursive=False)¶

Delete a specified file or directory. This function does not recursively delete by default.

Examples:
>>> rm('/tmp/build', recursive=True)
opsutils.run(command, **kwargs)¶

Run a shell command and wait for the response. The result object will resolve to True if result.code == 0 and output/error results can be retrieved from result.stdout and result.stderr variables.

Examples:
>>> run('ls ${path}', path='/tmp').code
0
opsutils.stat(path)¶

Access stat attributes of files and directories.

Examples:
>>> stat('/tmp').user.name
root
opsutils.user(id=None, name=None)¶

Helper class for getting information about a user.

Examples:
>>> user(id=0).name
root
opsutils.workspace(suffix='', prefix='tmp', dir=None)¶

Create a secure and temporary workspace that will be automatically cleaned up. workspace takes the same parameters as tempfile.mkdtemp.

Examples:
>>> with workspace('test') as w:
...     print w.join('dir1', 'file1')
